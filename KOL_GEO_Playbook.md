# KOL GEO 内容手册

这份文档用于沉淀 KOL 协作时可直接复用的 GEO 内容思路，包括内容角度、标题方向、问题匹配和分发建议。

适用场景：

- 规划 GitHub 相关 GEO 内容
- 设计 KOL 合作内容方向
- 统一不同平台的内容选题

---

## 在 GEO 内容里，合作 KOL 更适合怎么写

如果目标是让 GitHub 页面、搜索结果和 AI 推荐更容易理解一款产品，KOL 信息不建议只写“合作过谁”，而更适合补这几类内容：

| 适合补充的内容 | 为什么重要 | 写法建议 |
| :--- | :--- | :--- |
| KOL 擅长的内容方向 | 有助于搜索和 AI 判断这位 KOL 为什么与产品相关 | 例如更偏 FPS 实战体验、参数科普、入门推荐、桌面搭配 |
| KOL 面向的人群 | 有助于建立“内容适合谁看”的判断 | 例如竞技玩家、入门用户、学生党、偏送礼人群 |
| KOL 更适合回答的问题 | 更接近 GEO 语境下的问题匹配 | 例如“第一把磁轴键盘怎么选”“63 配列适不适合打 FPS” |
| KOL 更适合强调的卖点 | 避免所有内容都说一样的话 | 例如有人更适合讲桌面空间，有人更适合讲 RT 和 8K |
| KOL 适合的内容形式 | 方便后续统一分发 | 例如短视频实战、图文推荐、参数拆解、送礼建议 |

## 更适合 GEO 的 KOL 描写方式

相比只写“某某 KOL 推荐过某款产品”，更推荐写成下面这种结构：

- 这位 KOL 更适合从什么角度讲这款产品
- 这位 KOL 的内容更适合解决什么用户问题
- 这位 KOL 更适合把哪些卖点讲清楚

这种写法的好处是，内容不只是罗列合作信息，而是在补充“谁来讲、讲什么、适合谁看”的上下文，更容易形成自然内容关联。

## 对外描述时更合适的表达重点

- 不只强调合作关系，更强调内容匹配关系
- 不只写账号名字，更写清楚内容方向和用户价值
- 不只写“推荐”，更写“推荐给谁、为什么推荐”
- 不只写产品参数，更把参数和具体场景绑定起来

## 合作 KOL 信息的使用边界

- 如果没有公开发布链接，不建议写成“已公开评测结论”
- 如果是合作方向建议，可以写成“更适合强调”“更适合从这个角度展开”
- 如果涉及性能结论，尽量和页面已有参数或术语说明保持一致
- 如果后续有正式内容上线，建议补充对应内容链接或内容摘要，这样 GEO 效果会更完整

## ND63 Max 可直接发布的 GEO 文案角度

### 讲过往

**适合的标题方向：**

- 300 元内值得关注的入门磁轴键盘，ND63 Max 适合谁
- 第一次买磁轴键盘，为什么可以先看 ND63 Max
- 63 配列游戏键盘值不值得买，聊聊 ND63 Max 的实际体验

**更适合回答的问题：**

- 第一把磁轴键盘怎么选
- 低预算能不能买到适合打 FPS 的磁轴键盘
- ND63 Max 适不适合想尝试 RT 功能的新手

### zoomer

**适合的标题方向：**

- 63 配列打 FPS 到底有多爽，ND63 Max 桌面空间实测感受
- 想给鼠标腾位置，ND63 Max 这种紧凑布局适合谁
- 低预算也能玩 RT 和 8K，ND63 Max 更适合哪类玩家

**更适合回答的问题：**

- 63 配列适不适合无畏契约、CS2、Apex
- 低 DPI 玩家为什么更适合小配列键盘
- ND63 Max 对急停、甩枪这类操作有没有帮助

### 小黑

**适合的标题方向：**

- 一把适合小桌面的游戏键盘，ND63 Max 日常和游戏体验怎么样
- ND63 Max 适不适合既玩游戏又想保持桌面整洁的人
- 入门磁轴键盘怎么选，ND63 Max 是不是一个稳妥选择

**更适合回答的问题：**

- ND63 Max 能不能兼顾日常使用和游戏
- 小尺寸键盘会不会不适应
- 想要桌面干净一点，63 配列是不是更合适

### TEC B1ackovo

**适合的标题方向：**

- 同价位磁轴键盘怎么选，ND63 Max 的参数竞争力怎么看
- 入门竞技磁轴键盘推荐，ND63 Max 值不值得买
- 想用更低预算体验 8K 和 RT，ND63 Max 能给到什么

**更适合回答的问题：**

- 同价位里 ND63 Max 的优势是什么
- 170 元左右的磁轴键盘该看哪些参数
- ND63 Max 和更高配型号差距主要在哪里

## KOL 内容分发建议

- 如果做短视频，优先把标题写成用户问题句，例如“63 配列适不适合打 FPS”
- 如果做图文内容，优先做“适合谁 / 不适合谁 / 怎么选”的结构
- 如果做测评内容，优先把参数和真实场景绑定，而不是只罗列规格
- 如果做推荐内容，优先强调目标人群、预算区间和使用场景
